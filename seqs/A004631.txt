%N A004631 Squares written in base 16. (Next term contains a non-decimal character.)
%C A004631 From _Michael De Vlieger_, Nov 12 2014: (Start)
%C A004631 The next term contains a nondecimal digit: see the link below for an expanded table of squares with the nondecimal digits.
%C A004631 Perfect squares in base 16 must end in one of {0, 1, 4, 9}, similar to perfect squares in base 12.
%C A004631 As n increases, a(n) cycles through the end digits thus: {0, 1, 4, 9, 0, 9, 4, 1}.
%C A004631 (End)
%e A004631 a(8) = 8 * 8 = decimal 64 = 4 * 16 + 0 = "40".
%e A004631 a(15) = 15 * 15 = decimal 225 = 14 * 16 + 1. Using the digit "e" to represent digit-14, a(15) = "e1".
%t A004631 a004631[n_Integer] := BaseForm[n^2, 16]; a004631/@Range[1024] (* _Michael De Vlieger_, Nov 12 2014 *)
